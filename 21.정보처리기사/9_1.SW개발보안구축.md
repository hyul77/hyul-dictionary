# 9_1.SW 개발 보안 구축
## SW 개발 보안
### Secure SDLC
- SDLC에 보안강화를 위한 프로세스를 포함한 것
- CLASP : SDLC의 초기 단계에서 보안을 강화하기 위해 개발된 방법론
- SDL : 안전한 SW 개발을 위해 기존의 SDLC를 개선한 방법론
- Seven Touchpoints : SW 보안의 모범사례를 SDLC에 통합한 방법론

### SW 개발 보안 요소
- 보안 3대 요소 : 기밀성, 무결성, 가용성
- 기밀성 : 인가된 사용자에게만 접근 허용, 정보 전송 중 노출 되어도 데이터를 읽을 수 없음
- 무결성 : 인가된 사용자만 수정 가능
- 가용성 : 인가된 사용자는 언제라도 자원을 사용할 수 있음
- 인증 : 사용자가 합법적인 사용자인지 확인 (패스워드, 인증용 카드, 지문 검사)
- 부인 방지 : 데이터를 송수신한 자가 송수신 사실을 부인할 수 없도록 증거 제공

### 시큐어 코딩
- 구현단계에서 발생할 수 있는 보안 취약점들을 최소화하기 위해 보안 요소들을 고려해 코딩하는 것
- 안정성, 신뢰성 확보
- 보안 정책을 바탕으로 시큐어 코딩 가이드 작성, 교육 실시

### 세션 통제 관련 용오
- 세션 : 서버와 클라이언트의 연결 의미
- 세션 통제 : 세션의 연결과 연결로 인해 발생하는 정보 관리
- 세션ID : 서버가 클라이언트들을 구분하기 위해 부여하는 키
- 스레드 : 프로세스 내의 작업 단위로 시스템의 자원을 할당받아 실행
- 다중 스레드 : 두개이상의 스레드가 생성되어 동시 처리되는 다중 작업을 다중 스레드라 함
- URL Rewrite : 쿠키를 사용할 수 없는 환경에서 세션 ID전달을 위해 URL에 세션 ID를 포함시키는 것

### 입력 데이터검증 및 표현 관련 용어
- 입력데이터 검증 및 표현은 입력데이터로 인해 발생하는 문제들을 예방하기 위해 구현 단계에서 검증하는 것
- 경로 순회 : 경로 탐색을 할 때 사용하는 '/','\'등의 기호를 악용하여 허가되지 않은 파일에 접근하는 방식
- 스크립트 : 처리 절차가 기록된 텍스트로 스크립트 파일 확장자로 asp, jsp, php등이 있음

### SQL 삽입
- 웹 응용 프로그램에 SQL을 삽입하여 내부 DB 데이터를 유출 및 변조하고 관리자 인증을 위회하는 보안 약점
- 동적 쿼리에 사용되는 입력 데이터에 예약어 및 특수 문자가 입력되지 않게 필터링 되도록 설정하여 방지

### 크로스사이트 스크립팅(XSS)
- 웹페이지에 악의적인 스크립트를 삽입하여 방문자의 정보를 탈취, 비정상적인 기능 수행을 유발하는 보안 약점
- HTML 태그의 사용을 제한, 스크립트에 삽입되지 않도록 '<','>','&'등의 문자를 다른 문자로 치환해 방지

### 메모리 버퍼 오버플로
- 연속된 메모리 공간을 사용하는 프로그램에서 할당된 메모리의 범위를 넘어선 위치에서 자료를 읽거나 쓰려고 할 때 발생하는 보안 약점
- 버퍼 크기를 설정하고 설정된 범위의 메모리 내에서 올바르게 읽거나 쓸 수 있도록 해 방지

### 보안 기능 관련 용어
- 보안 기능은 SW 구현 단계에서 코딩하는 기능인 인증, 접근제어, 기밀성, 암호화등을 올바르게 구현하기 위한 점검 항목
- HTTPS : 웹과 서버간의 안전한 통신을 위해 HTTP와 암호 통신 규약을 결합한 것
- SSL : 데이터를 송수신하는 컴퓨터 사이 TCP/IP계층과 애플리케이션 계층 사이에 위치해 인증, 암호화, 무결성을 보장
- 하드코딩 : 데이터를 코드 내부에직접 입력하여 프로그래밍하는 방식

### 하드코드된 암호화키
- 암호화된 키도 하드코드된 경우 유출시 역계산 또는 무차별 대입 공격에 의해 탈취 가능
- 암호화 키 생성 모듈 또는 보안이 보장된 외부 공간을 이용해 방지

### TOCTOU 경쟁 조건
- 검사 시점과 사용시점을 고려하지 않고 코딩하는 경우 발생하는 보안 약점

### 널 포인터 역참조
- 널 포인터가 가리키는 메모리의 위치에 값을 저장할 때 발생하는 보안 약점
- 포인터를 이용하기 전에 널 값을 가지고 있는지 검사해 방지

### 스택 가드
- 주소가 저장되는 스택에서 발생하는 보안 약점을 막는 기술
- 복귀 주소와 변수 사이에 특정 값을 저장한 후 그 값이 변경되면 오버플로우 상태로 판단해 프로그램을 중단해서 잘못된 복귀 주소의 호출을 막음

### 접근 제어자
- 외부로부터 접근을 제한하기 위해 사용되는 예약어
- Public, Protected, Default, Private

## 키 관리
### 개인키 암호화 기법
- 동일한 키로 데이터를 암호화하고 복호화하는 암호화 기법
- 속도가 빠르나 관리해야하는 키의 수가 많음
- 스트림 암호화 : 평문과 동일한 길이의 스트림을 생성하여 비트 단위로 암호화(LFSR, RC4)
- 블록 암호화 : 한번에 하나의 데이터 블록을 암호화(DES, SEED, AES, ARIA)

### 공개키 암호화 기법
- 데이터를 암호화할 때 사용하는 공개키는 사용자에게 공개하고 복호화할 때의 비밀키는 관리자가 비밀리에 관리하는 암호화 기법
- 비대칭 암호 기법
- 속도가 느리지만 관리하는 키의 수가 적음
- RSA 기법이 대표적

### SEED
- KISA에서 개발, 블록 암호화 알고리즘
- 블록 크기 128비트, 키 길이에 따라 128, 256으로 분류

### ARIA
- 국가정보원과 산학연협회가 개발, 블록 암호화 알고리즘
- AES 암호화 알고리즘의 규격을 따름

### DES
- 미국 NBS에서 개발, 개인키 암호화 알고리즘
- 블록 크기 64비트, 키 길이 56비트, 16회의 라운드를 수행
- DES를 3번 적용하여 보안을 더욱 강화한 3DES가 있음

### AES
- 미국 NIST에서 개발, 개인키 암호화 알고리즘
- DES의 한계를 느껴 블록 크기 128비트, 키 길이에 따라 AES-128, AES-192, AES-256으로 분류

### RSA
- MIT에서 개발, 공개키 암호화 알고리즘 
- 큰 숫자를 소인수분해하기 어려운 걸 기반으로 만듦

### 해시
- 임의의 길이의 입력 데이터나 메시지를 고정된 길이의 값이나 키로 변환
- SHA시리즈, HAVAL, MD4, MD5, N-NASH, SNEFRU

### SHA 시리즈
- NSA이 설계 NIST 발표
- SHA-0, 1, 224, 256, 384, 512

### MD5
- MD4 대체하기 위해 고안된 암호화 해시 함수
- 블록 크기 512비트, 키 길이 128비트

### N-NASH
- 일본 NTT에서 발표, 암호화 해쉬 함수
- 블록 크기, 키 길이 128비트

### SNEFRU
- 해시함수
- 32비트 프로세서에서 구현이 용이하게 할 목적으로 개발

### TKIP
- 랜 보안 프로토콜인 WEP의 취약성을 보완한 데이터 보안 프로토콜
- 암호 알고리즘의 입력 키 길이를 128비트로 늘리고 패킷당 키할당, 키값 재설정 등 키 관리 방식 개선

## 보안
### 인증
- 다중 사용자 컴퓨터 시스템이나 네트워크 시스템에서 로그인 요청한 사용자의 정보를 확인하고 접근권한 검증 보안 절차
- 지식 기반 인증 : 고정된 패스워드, 패스 프레이즈, 아이핀
- 소유 기반 인증 : 신분증, 메모리카드, 스마트카드, OTP
- 생체 기반 인증 : 지문, 홍채/망막, 얼굴, 음성, 정맥
- 위치 기반 인증 : GPS, IP, 콜백
- 행위 기반 인증 : 서명, 동작

### 보안 아키텍처
- 무결성, 가용성, 기밀성을 확보하기 위해 보안 요소 및 보안 체계를 식별하고 관계를 정의
- 관리적, 물리적, 기술적 보안 개념의 수립, 보안 관리 능력의 향상, 일관된 보안 수준의 유지를 기대

### 보안 프레임워크 
- 안전한 정보 시스템 환경을 유지하고 보안 수준을 향상시키기 위한 체계
- ISO 27001 : 국제 표준, 대표적인 보안 프레임워크

### 방화벽
- 내부의 네트워크와 인터넷 간에 전송되느니 정보를 선별하여 수용, 거부, 수정하는 기능을 가진 침입 차단 시스템
- 외부로 나가는 패킷은 그대로 통과, 내부로 오는 패킷은 내용을 엄밀히 체크, 인증된 패킷만 통과

### 침입 탐지 시스템
- IDS는 컴퓨터 시스템의 비정상적인 사용, 오용, 남용 등을 실시간으로 탐지
- 오용탐지 : 미리 입력해둔 공격 패턴이 감지되면 알림
- 이상탐지 : 평균적인 시스템의 상태를 기준으로 비정상적인 행위나 자원의 사용이 감지되면 알려줌

### 침입 방지 시스템
- IPS 는 비정상적인 트래픽을 능동적으로 차단하고 격리하는 등의 방어 조치를 취하는 보안 솔류션
- 침입 탐지 기능으로 패킷을 하나씩 검사후 비정상적인 패킷 탐지 시 방화벽 기능으로 해당 패킷을 차단

### 데이터 유출 방지
- 내부 정보의 외부 유출을 방지하는 보안 솔루션
- 사내 직원이 사용하는 PC와 네트워크상의 모든 정보를 검색하고 메일, 메신저, 웹하드, 네트워크 프린터 등의 사용자 행위를 탐지, 통제해 외부로의 유출을 사전에 막음

### 웹 방화벽
- 일반 방화벽이 탐지 못하는 웹 기반 공격을 방어할 목적으로 만들어진 웹서버에 특화된 방화벽
- 감시, 도달 전에 차단

### VPN
- 공중 네트워크와 암호화 기술을 이용하여 사용자가 마치 자신의 전용 회선을 사용하는 것 처럼 해주는 보안 솔루션
- 암호화된 규격을 통해 인터넷 망을 전용선의 사설망을 구축한 것 처럼 이용, 비용 부담을 줄임

### NAC
- 네트워크에 접속하는 내부 PC의 MAC 주소를 IP 관리 시스템에 등록한 후 일관된 보안 관리 기능을 제공하는 보안 솔루션
- 내부 pc의 SW 사용 현황을 관리하여 불법적인 SW 설치 방지

### ESM
- 다양한 장비에서 발생하는 로그 및 보안 이벤트를 통합하여 관리하는 보안 솔루션
- 보안 이벤트를 통합하여 관리해 비용 및 자원을 절약
- 보안 솔루션 간의 상호 연동을 통해 종합적인 보안 관리 체계 수립

### AAA(3A)
- 3가지 기능을 기반으로 자원 접근에 대한 처리와 서비스를 제공 인증, 인가, 과금
- 인증 : 접근하는 사용자의 산원 검증
- 인가 : 신원이 검증된 사용자에게 특정 권한, 서비스 허용
- 과금 : 사용자가 어떤 종류의 서비스를 이용하고 얼마만큼의 자원을 사용했는지 기록 및 보관

### 정보보호관리체계
- ISMS : 정보 자산을 안전하게 보호하기 위한 보호 절차와 대책을 의미
- 한국인터넷진흥원(KISA)

### 서비스 거부 공격
- DoS : 대량의 데이터를 한 곳의 서버에 집중적으로 전송함으로써 표적이 된 서버의 정상적인 기능을 방해하는 것
- Ping of Death, SMURFING, SYN Flooding, TearDrop, LAND Attack, DDoS 공격

### Ping of Death
- 패킷 크기를 허용 범위 이상으로 전송하여 네트워크를 마비시키는 서비스 거부 공격 방법

### SMURFING
- IP나 ICMP의 특성을 악용하여 엄청난 양의 데이터를 한 사이트에 집중적으로 보내 네트워크를 불능 상태로 만듦
- 각 네트워크 라이터에서 브로드캐스트 주소를 사용할 수 없게 미리 설정

### SYN Flooding
- TCP는 신뢰성 있는 전송을 위해 3-wayhandshake를 하고 전송하는데 공격자가 가상의 클라이언트로 위장해 3-wy-handshake 과정을 의도적으로 중단 시킴

### TearDrop
- 데이터의 송수신 과정에서 패킷의 크기가 커 여러개로 분할되어 전송될 때 분할 순서를 알 수 있게 Fragment Offset 값을 함께 전송함
- TearDrop은 Offset 값을 변경시켜 수신 측에서 패킷을 재조립할 때 과부하 발생, 시스템 다운되도록 하는 공격 방법

### LANDAttack
- 패킷을전송할 때 송신 IP 주소와 수신 IP 주소를 모두 공격 대상의 IP 주소로 하여 공격 대상에게 전송하는 것
- 자신에 대해 무한히 응답하게 하는 공격방법

### DDos 공격
- 여러곳에 분산된 공격 지점에서 한 곳의 서버에 대해 서비스 거부 공격을 수행하는 것

### 분산 서비스 공격용 툴
- TrinOO : 초기 형태의 데몬, 주로 UDP Flooding 공격을 수행
- TFN : UDP 뿐만아니라 TCP SYN Flood 공격, ICMP 응답 요청, 스머핑 공격 수행
- TFN2K : TFN 확장판
- Stacheldraht : 공격자, 마스터, 에이전트가 쉽게 노출되지 않도록 암호화된 통신 수행, 툴이 자동으로 업데이트 됨

### 세션 하이재킹
- 상호 인증 과정을 거친 후 접속해 있는 서버와 클라이언트 사이의 세션 정보를 가로채는 공격 기법
- TCP 3-Way-Handshake 과정에 끼어들어 동기화된 시퀀스 번호를 가로채 서버에 무단으로 접근하는 TCP 세션 하이재킹이 대표적

### ARP 스푸핑
- 자신의 물리적 주소(MAC)를 공격 대상의 것으로 변조하여 데이터 패킷을 가로채거나 방해하는 기법
- ARP : 호스트의 IP 주소를 호스트와 연결된 네트워크 접속 장치의 물리적 주소(MAC)로 변환해주는 프로토콜

### 스니핑
- 네트워크의 중간에서 남의 패킷 정보를 도청하는 해킹 유형의 하나로 수동적 공격에 해당

### 워터링 홀
- 목표 대상이 자주 방문하는 웹 사이트를 사전에 감염시켜 대상이 해당 사이트에 방문했을 때 악성 코드에 감염되게 하는 웹 기반 공격
- 감염된 PC를 기반으로 대상이 속한 조직의 중요 시스템에 접근하거나 불능으로 만드는 등의 영향력을 행사

### 키로거 공격
- 컴퓨터 사용자의 키보드 움직임을 탐지해 ID, 패스워드, 계좌번호, 카드번호 등과 같은 개인 정보를 빼가는 해킹 공격

### 랜섬웨어
- 내부 문서나 파일 등을 암호화해 사용자가 열지 못하게 하는 프로그램, 암호 해독용 프로그램의 전달을 조건으로 돈을 요구

### 백도어
- 시스템 설계자가 서비스 기술자나 유지보수 프로그램 작성자의 액세스 편의를 위해 시스템 보안을 제거하여 만들어놓은 비밀 통로
- 백도어 탐지방법 : 무결성 검사, 열린 포트 확인, 로그 분석, SetUID 파일 검사

### 보안용어 모음
- 스미싱 : 문자를 통해 신용 정보 뺴냄
- 스피어 피싱 : 일반적인 이메일로 위장, 발송 메일의 본문 링크나 첨부된 파일 클릭해 개인 정보 탈취
- APT : 특정 기업이나 조직 네트워크에 침투해 활동 거점을 마련한 후 보안을 무력화 시키고 정보 수집후 빼돌림
- 무작위 대입 공격 : 암호화된 문서의 암호키를 찾아내기 위해 적용 가능한 모든 값을 대입
- 큐싱 : QR코드를 통해 악성 앱의 단로드를 유도하는 금융사기 기법
- WEP : 유선 랜에서 기대할 수 있는 것과 같은 보안과 프라이버시 수준의 무선 랜의 보안 포로토콜
- WPA2 : WEP의 취약성에 대한 대안으로 발표된 무선랜 보안 기술 규격
- EDR : 엔드 포인트 영역에 대한 지속적인 모니터링을 통해 행위 기반 위협 탐지 및 분석, 대응 기능을 제공하는 솔루션
- TMS : 전사적 IT 인프라에 대한 위협정보들을 수집분석경보관리하는 정보보호 통합관리 시스템
- DTLS : UDP 기반으로 통신하는 SSL/TLS와 유사한 보안 기능을 제공하는 프로토콜
- 크리덴셜 스터핑 : 무차별 대입 공격의 일종으로 공격자가 미리 확보해놓은 로그인 자격 증명을 다른 계정에 무작위로 대입
- POODLE 공격 : TLS 연결을 SSL3.0으로 낮춰 취약점을 이용해 암호문을 해독
- DDE : 엑셀 등 오피스 프로그램에서 DDE가 활성화 될 시 일반 문서로 위장한 악성 코드가 전파 될 수 있음
- 좀비PC : 악성 코드에 감염되어 컴퓨터 조종하도록 만들어진 컴퓨터
- C&C 서버 : 원격지에서 감염된 좀비 PC에 명령을 내리고 악성코드를 제어하기 위한 용도로 사용되는 서버
- 봇넷 : 악의적인 의도로 사용될 수 있는 다수의 컴퓨터들이 네트워크로 연결 된 상태
- 웜 : 연속적으로 자신을 복제해 시스템의 부하를 높여 시스템을 다운 시키는 바이러스
- 제로 데이 공격 : 보안 취약점이 발견되었을 때 발견된 취약점의 존재 자체가 널리 공표되기 전에 해당 취약점을 통해 이루어지는 보안 공격(공격의 신속성의미)
- 트로이 목마 : 정상적인척 위장하여 숨어있다가 동작할 때 활성화되어 부작용을 일으킴
- CC 인증 : 국가마다 서로 다른 정보보호 시스템 평가기준을 연동하고 평가결과를 상호 인증하기 위해 제정된 정보보안 평가 기준
- 멀버타이징 : 멀웨어와 광고의 합성어로 온라인 광고를 통해 악성코드 유포
- 정보공유분석센터 : 침해사고가 발생하는 경우 실시간 경보 분석체계를 운영하고 금융 통신 등 분야별 정보통신 기반 시설을 보호하기 위한 업무 수행
- 업무 연속성계획 : 위기관리를 기반으로 재해복구 업무 복구 및 재개 비상계획 등의 비즈니스 연속성 보장하기 위한 계획
- 데이터 디들링 : 처리할 자료를 다른 자료와 바꿔서 처리하는 것
- 비트로커 : Window 전용의 볼륨 암호화 기능 TPM과 AES-128 알고리즘 사용
- 공급망 공격 : SW 공급망에 치투해 악성 코드를 배포
